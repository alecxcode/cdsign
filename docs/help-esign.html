Digitally sign a document: how to use the app

Introduction

A digital signature is a mathematical scheme for verifying the authenticity of electronic documents. With this app it uses secret and public keys, and detached digital signature file.
With secret key you can sign an electronic document, secret key should not be transferred to any other person. Public key should be given to another person in order they can verify your signature and document integrity.
When you sign a file with a digital signature a hash is being generated, which then modified by cryptographic algorithms. This modified hash is the digital signature of a specific electronic document.
All this is essential in order to verify (using public key and that hash) that a a specific electronic document signed by a person specified in that public key and not modified during transmission (integrity check).
Although, this program does not utilize any certificate authority (an entity that issues digital certificates).
This application itself does not have any cryptographic algorithms, for this purpose it utilizes another program - GPG. This app is basically simplified graphical user interface for GPG.
Public key and digital signature files, generated with the application you can send to another person so they can verify your signature and an electronic document integrity with Check document digital signature application.

Create digital signature keys

In order to generate digital signature keys:
  1. Launch the application
  2. Press the button Create DS keys...
  3. Input the data requested
  4. Press the button Generate keys
  5. Wait until the process of generation ends
  6. The application should display the result
The key you created will be shown next time you launch the app in the main window. Public and Secret keys are stored in a keyring of the GPG program.

Export a public key to give to another person

To export a public key you just press Export public key... button and select or enter filename to save the key to.
The file will have an extension of .key. The application should display the result of export.
It is necessary to send this public key to other parties you interact with. They will need it to check your digital signature and integrity of electronic documents.

Sign a file with a digital signature

In order to sign a file with a digital signature:
  1. Launch the application
  2. Press the button Document (or contract) file...
  3. Select a file to sign
  4. Press the button Sign the file
  5. The application should display the result
Digital signature file generated with the app (you can find in a directory with a signed file, and it usually has an extension of .asc) you need to send to another party with a signed file. Using these files and a public key they can check your digital signature and integrity of electronic documents.

Application settings (for advanced users)

The settings are: (1) GPG executable name; (2) last generated key ID.
The settings are stored in: user_home_directory/.edm/cdsign-esign.ini
